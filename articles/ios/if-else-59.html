Swift 5.9's Streamlined Code with Enhanced If and Switch Expressions Swift 5.9 If and Switch Expressions Published on Sep 12, 2023

The release of Swift 5.9, developers can now enjoy an enhanced coding experience, thanks to the introduction of improved if and switch expressions. These new features allow for the omission of the return keyword, resulting in code that is not only shorter but also significantly more readable. In this article, we'll dive into the Swift 5.9 release and explore these exciting additions through a new example.

Utilizing If Statements for Clarity

Let's explore this new feature using an example. Consider a common scenario: calculating the discounted price of an item based on whether a user has a premium membership. In previous versions of Swift, you might have written code like this: 

var discountedPrice: Double
if hasPremiumMembership {
    discountedPrice = regularPrice * 0.9
} else {
    discountedPrice = regularPrice
}

This code is functional but spans multiple lines. While it might not seem overly long, combining numerous such statements can quickly lead to unwieldy functions. Some developers might opt for the ternary operator to shorten the code: 

var discountedPrice: Double = hasPremiumMembership ? regularPrice * 0.9 : regularPrice

However, the ternary operator isn't universally embraced for its readability.

Swift 5.9: The Solution

Swift 5.9 introduces a groundbreaking feature: the ability to use if statements as expressions. This innovation allows you to write more concise code while preserving readability.

Here's how it works:

var discountedPrice: Double = if hasPremiumMembership { regularPrice * 0.9 } else { regularPrice }

This new syntax, introduced in Swift 5.9, closely mirrors the practice of omitting the return keyword inside closures. It streamlines your code and simplifies the process of writing conditional expressions.

However, it's important to note that the if statement branch must contain a single expression , and multiple lines of code within a single branch are not permitted. Additionally, using if statements as expressions requires the presence of an else branch .

The Enhanced Power of Switch Statements

Swift developers frequently use switch statements to handle various cases and return values based on enumeration values.

Traditionally, you might have written code like this:

enum DayOfWeek {
    case monday, tuesday, wednesday, thursday, friday, saturday, sunday
    var isWeekend: Bool {
        switch self {
        case .saturday, .sunday:
            return true
        default:
            return false
        }
    }
}

This code is clear, but the return keyword on a new line for each case can feel somewhat redundant. Prior to Swift 5.9, developers could reduce code length by omitting the return keyword: 

var isWeekend: Bool {
    switch self {
    case .saturday, .sunday: return true
    default: return false
    }
}
Swift 5.9: The Solution

With Swift 5.9, the language takes this simplification a step further: 

var isWeekend: Bool {
    switch self {
    case .saturday, .sunday: true
    default: false
    }
}

By eliminating the need for the return keyword altogether, Swift 5.9 empowers you to craft more concise and elegant switch expressions.

Conclusion

Swift 5.9's introduction of if and switch expressions is a remarkable advancement for developers aiming to write cleaner, more efficient code. These expressions allow you to omit the return keyword, resulting in code that is not only shorter but also significantly more natural to read.
